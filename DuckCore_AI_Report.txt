ABSTRACT

DuckCore AI is an advanced AI-powered code assistance platform designed to streamline the software development process. The application leverages large language models (LLMs) via the OpenRouter API to provide developers with instant code explanations, debugging assistance, and performance optimization recommendations. Built with the MERN stack (MongoDB, Express, React, Node.js), DuckCore AI offers a seamless user experience for analyzing code snippets across various programming languages. The system incorporates a history tracking mechanism, allowing users to revisit previous analyses and maintain a log of their debugging sessions. This report details the design, implementation, and architectural aspects of DuckCore AI, highlighting its role in enhancing developer productivity and code quality.

CHAPTER-1

1.1 INTRODUCTION
DuckCore AI is a comprehensive tool designed for developers, students, and engineers to interact with code more intelligently. As software complexity grows, the need for automated tools that can interpret and improve code becomes critical. DuckCore AI fills this gap by providing a centralized interface where users can input code and receive tailored AI-generated feedback.

1.2 OBJECTIVES
The primary objectives of the DuckCore AI project are:
- To provide a user-friendly interface for code analysis.
- To implement AI-driven code explanation, debugging, and optimization.
- To store and manage a history of user interactions for future reference.
- To ensure high performance and low latency in AI response generation.

1.3 PROJECT INTERFACE
The project interface consists of a responsive web application featuring a code editor input area, a selection menu for analysis types (Explain, Debug, Optimize), and a clear output display for AI results. Navigation is facilitated through a sidebar and navbar, allowing easy access to history and user profiles.

CHAPTER-4

4.1 PROBLEM DESCRIPTION
The core challenge addressed by DuckCore AI is the cognitive load associated with code maintenance. By providing a tool that "understands" code, we reduce the time spent on debugging and increase the time spent on feature development.

4.2 CODE ANALYSIS MODULE
This module handles the core functionality of the application. It processes three types of requests:
- Explain: Breaks down code logic into human-readable steps.
- Debug: Identifies logical or syntax errors and provides fixes.
- Optimize: Suggests better algorithmic approaches or cleaner syntax.

4.3 HISTORY MANAGEMENT MODULE
To prevent the loss of insights, this module automatically saves every analysis performed by the user. Users can search through their history, view previous code snippets, and see the results provided by the AI at that time.

9.1 CODING

Key Backend Logic (server.js):
const generateResponse = async (type, code, language) => {
  let prompt = "";
  if (type === "explain") {
    prompt = `Explain the following ${language} code concisely...`;
  } else if (type === "debug") {
    prompt = `Debug the following ${language} code...`;
  } else if (type === "optimize") {
    prompt = `Optimize the following ${language} code...`;
  }
  const response = await axios.post(
    "https://openrouter.ai/api/v1/chat/completions",
    {
      model: "anthropic/claude-3-haiku",
      messages: [{ role: "user", content: prompt }]
    },
    { headers: { "Authorization": `Bearer ${process.env.OPENROUTER_API_KEY}` } }
  );
  return response.data.choices[0].message.content;
};

Mongoose History Schema (History.js):
const historySchema = new mongoose.Schema({
  code: { type: String, required: true },
  result: { type: String, required: true },
  type: { type: String, enum: ["explain", "debug", "optimize"] },
  date: { type: Date, default: Date.now }
});
